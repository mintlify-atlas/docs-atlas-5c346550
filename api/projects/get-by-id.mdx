---
title: 'Get Project by ID'
description: 'Retrieve a specific photography project'
---

## Endpoint

```
GET /api/projects/{id}
```

## Authentication

This endpoint does not require authentication. It is publicly accessible.

## Description

Retrieves a specific photography project by its unique identifier, including all associated detail images.

## Path Parameters

<ParamField path="id" type="string (GUID)" required>
  The unique identifier of the project to retrieve.
  
  Format: `3fa85f64-5717-4562-b3fc-2c963f66afa6`
</ParamField>

## Response

<ResponseField name="id" type="string (GUID)">
  Unique identifier for the project.
</ResponseField>

<ResponseField name="title" type="string">
  Project title.
</ResponseField>

<ResponseField name="description" type="string">
  Project description.
</ResponseField>

<ResponseField name="imageUrl" type="string">
  URL of the main project image/cover.
</ResponseField>

<ResponseField name="isActive" type="boolean">
  Indicates if the project is currently active.
</ResponseField>

<ResponseField name="createDate" type="string (datetime)">
  ISO 8601 timestamp when the project was created.
</ResponseField>

<ResponseField name="details" type="array">
  Array of project detail objects containing additional images.
  
  <Expandable title="ProjectDetailDto properties">
    <ResponseField name="id" type="string (GUID)">
      Unique identifier for the detail.
    </ResponseField>
    
    <ResponseField name="imageUrl" type="string">
      URL of the detail image.
    </ResponseField>
    
    <ResponseField name="isActive" type="boolean">
      Indicates if the detail is currently active.
    </ResponseField>
  </Expandable>
</ResponseField>

## Example Request

```bash
curl -X GET http://localhost:5000/api/projects/3fa85f64-5717-4562-b3fc-2c963f66afa6
```

### Error (404 Not Found)

Returned when no project exists with the specified ID:

```json
{
  "message": "No se encontró el proyecto con Id 3fa85f64-5717-4562-b3fc-2c963f66afa6"
}
```

### Error (400 Bad Request)

Returned when the ID format is invalid (not a valid GUID):

```json
{
  "type": "https://tools.ietf.org/html/rfc7231#section-6.5.1",
  "title": "One or more validation errors occurred.",
  "status": 400,
  "errors": {
    "id": [
      "The value 'invalid-id' is not valid."
    ]
  }
}
```

## Notes

- The ID must be a valid GUID format
- Returns the project regardless of its `isActive` status
- Includes all associated detail images in the response
- No authentication required - this is a public endpoint
