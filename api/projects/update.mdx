---
title: 'Update Project'
description: 'Update an existing photography project'
---

## Endpoint

```
PUT /api/projects/{id}
```

## Authentication

**Required**: Admin role

This endpoint requires a valid JWT token with the `Admin` role. Include the token in the `Authorization` header:

```
Authorization: Bearer YOUR_JWT_TOKEN
```

## Description

Updates an existing photography project and its detail images. The project ID in the URL must match the ID in the request body.

## Path Parameters

<ParamField path="id" type="string (GUID)" required>
  The unique identifier of the project to update.
  
  Format: `3fa85f64-5717-4562-b3fc-2c963f66afa6`
</ParamField>

## Request Body

<ParamField body="id" type="string (GUID)" required>
  Project ID. Must match the ID in the URL path.
</ParamField>

<ParamField body="title" type="string" required>
  Project title. Maximum length: 200 characters.
</ParamField>

<ParamField body="description" type="string">
  Project description. Maximum length: 1000 characters. Optional.
</ParamField>

<ParamField body="imageUrl" type="string" required>
  URL of the main project cover image. Maximum length: 500 characters.
</ParamField>

<ParamField body="bannerClickTitle" type="string">
  Title displayed when the project banner is clicked. Maximum length: 200 characters. Optional.
</ParamField>

<ParamField body="bannerClickDescription" type="string">
  Description displayed when the project banner is clicked. Maximum length: 1000 characters. Optional.
</ParamField>

<ParamField body="isActive" type="boolean" default="true">
  Whether the project is active and visible. Defaults to `true`.
</ParamField>

<ParamField body="details" type="array" default="[]">
  Array of detail objects containing project images. This replaces all existing details.
  
  <Expandable title="ProjectDetailCreateDto properties">
    <ParamField body="imageUrl" type="string" required>
      URL of the detail image. Maximum length: 500 characters.
    </ParamField>
    
    <ParamField body="isActive" type="boolean" default="true">
      Whether the detail is active. Defaults to `true`.
    </ParamField>
  </Expandable>
</ParamField>

## Response

<ResponseField name="id" type="string (GUID)">
  Unique identifier for the project.
</ResponseField>

<ResponseField name="title" type="string">
  Project title.
</ResponseField>

<ResponseField name="description" type="string">
  Project description.
</ResponseField>

<ResponseField name="imageUrl" type="string">
  URL of the main project image.
</ResponseField>

<ResponseField name="isActive" type="boolean">
  Indicates if the project is active.
</ResponseField>

<ResponseField name="createDate" type="string (datetime)">
  ISO 8601 timestamp when the project was originally created.
</ResponseField>

<ResponseField name="details" type="array">
  Array of project detail objects.
  
  <Expandable title="ProjectDetailDto properties">
    <ResponseField name="id" type="string (GUID)">
      Unique identifier for the detail.
    </ResponseField>
    
    <ResponseField name="imageUrl" type="string">
      URL of the detail image.
    </ResponseField>
    
    <ResponseField name="isActive" type="boolean">
      Indicates if the detail is active.
    </ResponseField>
  </Expandable>
</ResponseField>

## Notes

- Admin role required - obtain token via [/api/auth/login](/api/auth/login)
- The project ID in the URL path **must** match the `id` field in the request body
- Updating a project replaces all existing detail images with the new details provided
- The `createDate` remains unchanged; only `modifiedDate` is updated internally
- Update operations are logged with the project ID for audit purposes
- To remove all detail images, pass an empty `details` array
- The `isActive` flag can be used to soft-delete projects without removing them from the database
