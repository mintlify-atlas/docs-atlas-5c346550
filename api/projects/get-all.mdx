---
title: 'Get All Projects'
description: 'Retrieve all active photography projects'
---

## Endpoint

```
GET /api/projects
```

## Authentication

This endpoint does not require authentication. It is publicly accessible.

## Description

Retrieves all active photography projects. Each project includes its details (images) as a nested collection.

## Response

Returns an array of project objects.

<ResponseField name="id" type="string (GUID)">
  Unique identifier for the project.
</ResponseField>

<ResponseField name="title" type="string">
  Project title.
</ResponseField>

<ResponseField name="description" type="string">
  Project description.
</ResponseField>

<ResponseField name="imageUrl" type="string">
  URL of the main project image/cover.
</ResponseField>

<ResponseField name="isActive" type="boolean">
  Indicates if the project is currently active.
</ResponseField>

<ResponseField name="createDate" type="string (datetime)">
  ISO 8601 timestamp when the project was created.
</ResponseField>

<ResponseField name="details" type="array">
  Array of project detail objects containing additional images.
  
  <Expandable title="ProjectDetailDto properties">
    <ResponseField name="id" type="string (GUID)">
      Unique identifier for the detail.
    </ResponseField>
    
    <ResponseField name="imageUrl" type="string">
      URL of the detail image.
    </ResponseField>
    
    <ResponseField name="isActive" type="boolean">
      Indicates if the detail is currently active.
    </ResponseField>
  </Expandable>
</ResponseField>

## Example Request

```bash
curl -X GET http://localhost:5000/api/projects
```

## Notes

- This endpoint returns only active projects (where `isActive` is `true`)
- Each project includes all its associated detail images
- Results are not paginated
- No authentication required - this is a public endpoint for displaying photography portfolio
